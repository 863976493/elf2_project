# Strawberry Tracker - 草莓视觉伺服追踪系统

## 项目概述

基于亚博 M3 PRO 机器人（Jetson Orin NX 8G）的草莓巡检视觉伺服系统。  
机器人在园区巡检过程中，通过 YOLO 检测 + 深度相机定位草莓束，自主靠近并稳定停车，为后续的拍照推理（成熟度 + 病害）做准备。

**当前阶段目标**：找到草莓束 → 靠近 → 稳定停车。拍照推理功能暂未接入，待追踪稳定后再加。

## 核心架构

### ROS2 话题

```
订阅：
  /camera/color/image_raw   (sensor_msgs/Image)   ← Orbbec RGB
  /camera/depth/image_raw   (sensor_msgs/Image)   ← Orbbec 深度图 (16-bit mm)

发布：
  /cmd_vel                  (geometry_msgs/Twist)  → 底盘控制
  /arm6_joints              (arm_msgs/ArmJoints)   → 机械臂关节角度
```

### QoS 策略

相机话题使用 **BestEffort + Volatile + KeepLast(1)**，与 Orbbec 驱动匹配。  
RGB 和 Depth 通过 `ApproximateTimeSynchronizer(queue_size=2, slop=0.1s)` 同步。

> **关键设计决策**：不使用点云（PointCloud2），直接从对齐后的深度图读取像素距离值，避免点云解析的巨大 CPU/带宽开销。

## 三段状态机

```
SEARCH ──检测到目标──→ APPROACH ──水平+距离到位──→ SETTLE
  ↑                      │                          │
  │                      │ 丢失超时                  │ 稳定完成 / 丢失超时 / 漂移
  └──────────────────────┴──────────────────────────┘
```

| 状态 | 行为 | 进入条件 | 退出条件 |
|------|------|----------|----------|
| **SEARCH** | 匀速旋转 (`search_angular_speed`) | 初始 / 目标丢失超时 / settle 完成 | 检测到目标 → APPROACH |
| **APPROACH** | PID 视觉伺服：水平旋转 → 前进靠近 → 机械臂垂直跟踪 | 从 SEARCH 检测到目标 | 水平误差 < deadband 且 距离误差 < deadband → SETTLE |
| **SETTLE** | 零速度，等待 `settle_time` 秒画面稳定 | 从 APPROACH 到位 | 稳定完成 → SEARCH；目标漂移超 2x deadband → APPROACH |

> **未来扩展点**：SETTLE 完成后将接入 CAPTURE 状态（拍照 + 成熟度/病害推理 + 上传云端）。当前 SETTLE 完成后直接回 SEARCH 继续巡检。

## 目标选择策略（ByteTrack + ID 锁定）

使用 `model.track(persist=True)` 替代 `model.predict()`，ByteTrack 为每个检测框分配**跨帧持久 ID**。

**选择逻辑**（`_select_target` 方法）：
1. **已锁定 ID 仍在** → 继续跟踪（最稳定）
2. **已锁定 ID 丢失** → 选离上次平滑位置最近的目标，重新锁定
3. **无先验信息** → 选面积最大的目标

> 关键：目标丢失时不立即清除 `_locked_id`，给 ByteTrack 几帧恢复窗口。只有超过 `no_detection_timeout`(1.5s) 才彻底清除并回到 SEARCH。

## EMA 平滑

对质心 (X, Y) 和深度值分别应用**指数移动平均**：

```
smooth = α × raw + (1 - α) × prev_smooth
```

- `ema_alpha = 0.35`（yaml 可调）
- 所有 PID 输入和机械臂控制都使用平滑后的值
- 目标丢失超时后重置平滑状态

**OSD 可视化**：黄色小圆 = 原始质心，绿色大圆 = 平滑质心，便于调试。

---

## PID 控制

三路独立 PID：

| 控制轴 | 输入误差 | 输出 | 参数 |
|--------|---------|------|------|
| 水平旋转 | 像素误差 (tgt_x - smooth_cx) | angular.z (rad/s) | kp=0.005, ki=0, kd=0.003 |
| 前进后退 | 深度误差 (tgt_dist - smooth_d) | linear.x (m/s) | kp=0.5, ki=0, kd=0.1 |
| 机械臂俯仰 | 像素误差 (smooth_cy - tgt_y) | joint4 角度 | gain=0.02 (仅 P 控制) |

**控制顺序**：先对准水平（angular deadband 5px 内），再前进/后退到目标距离（distance deadband 0.05m 内）。

## 深度读取

从深度图（非点云）直接读取：
1. 取平滑质心像素坐标 (u, v)
2. 提取 5x5 ROI 区域
3. 过滤无效值（0 或 NaN）
4. 取中值作为当前原始深度
5. 经 EMA 平滑后输出给 PID

> 深度值 > 100 视为毫米单位，除以 1000 转米；否则视为已经是米。

## 关键参数速查（tracker_params.yaml）

| 参数 | 默认值 | 说明 | 调参建议 |
|------|--------|------|----------|
| `confidence_threshold` | 0.45 | YOLO 置信度阈值 | 漏检多则降低，误检多则提高 |
| `ema_alpha` | 0.35 | EMA 平滑系数 | 还抖就降到 0.2~0.25（更平滑但响应变慢） |
| `settle_time` | 0.8 | SETTLE 等待秒数 | 画面稳定快可降到 0.5 |
| `target_distance` | 0.35 | 目标停车距离(m) | 根据相机焦距和草莓大小调整 |
| `angular_deadband` | 5 | 水平对准死区(px) | 太灵敏则增大 |
| `distance_deadband` | 0.05 | 距离对准死区(m) | 太灵敏则增大 |
| `no_detection_timeout` | 1.5 | 丢失超时(s) | ByteTrack 1.5s 足够恢复 |
| `max_angular_speed` | 0.8 | 最大旋转速度(rad/s) | 底盘响应慢可降低 |
| `max_linear_speed` | 0.3 | 最大前进速度(m/s) | 底盘响应慢可降低 |
| `arm_pitch_gain` | 0.02 | 机械臂俯仰增益 | 臂抖动则降低 |

## 已解决的问题

### 1. 草莓束质心跳变
- **原因**：原方案用 `model.predict()` 单帧检测 + 多框聚类合并，帧间检测数量波动导致合并框和质心剧烈跳变
- **方案**：`model.track(persist=True)` 引入 ByteTrack 跨帧跟踪 + ID 锁定 + EMA 平滑质心

### 2. 深度相机响应慢
- **原因**：Subscriber 使用默认 Reliable QoS，与相机驱动 BestEffort 不匹配，导致帧堆积或丢失
- **方案**：显式使用 BestEffort + Volatile + KeepLast(1) QoS；同步器 queue_size 从 5 降到 2 减少缓冲

### 3. 近距离视觉伺服抖动
- **原因**：靠近草莓后视野变小，微小的检测偏差被放大为底盘运动
- **方案**：三段状态机，APPROACH 到位后立即进入 SETTLE 零速等待，不做近距离微调
