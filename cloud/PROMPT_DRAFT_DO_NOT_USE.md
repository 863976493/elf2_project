# 项目背景
我正在开发一个“草莓种植园智能巡检机器人系统”。目前网络环境为同一局域网（未来可能做内网穿透）。
硬件架构如下：
1. **云端中控（伪云端）**：一台局域网内的 PC，作为数据中心、Web 服务器和控制台。
2. **边缘计算节点（小车大脑）**：RK3588，负责 ROS 2D 建图、Nav2 导航、以及基于 YOLO 的作物本地推理（虫害+成熟度），并控制摄像头。
3. **传感器节点**：多个 ESP32，分布在不同园区，负责驱动温湿度传感器。

# 功能需求
请帮我设计并编写“PC 云端部分”的系统架构和核心代码。云端需要实现以下 5 个核心功能：
1. **地图与路径规划交互**：接收 RK3588 传来的 2D 园区全景图并展示在 Web 端。支持在二维地图上直观地手动绘制/点选巡检路线，并将路径坐标下发给小车。
2. **环境数据监控**：接收并展示各个园区 ESP32 采集的热传感和温湿度数据。
3. **策略下发与阈值设置**：
   - 智能触发：可在网页端设置温湿度阈值，一旦 ESP32 数据越限，云端自动下发指令，调度小车前往异常园区。
   - 定时任务：可为每个园区设定定时巡检时间，到点自动下发巡检指令。
4. **多媒体与推理结果展示**：接收并存档 RK3588 上传的 YOLO 推理结果（如异常状态、成熟度）及附加的现场照片/短视频。支持开启小车的实时摄像头监控画面。
5. **大模型决策支持**：云端接入 LLM API（如 OpenAI/Claude API），定期将 ESP32 收集的数据和小车的巡检结果汇总发送给大模型，获取种植建议和数据分析。
